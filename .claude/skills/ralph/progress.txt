## Codebase Patterns
- Grafana dashboards go in `observability/grafana/dashboards/*.json` â€” setup.sh auto-creates ConfigMaps with `grafana_dashboard=1` label from all JSON files in that directory
- Dashboard JSON follows schemaVersion 39, uses `{ "type": "prometheus", "uid": "prometheus" }` as datasource
- Existing dashboards: kcp-resources.json (uid: kcp-resources), kcp-server-health.json (uid: kcp-server-health), kcp-etcd-health.json (uid: kcp-etcd-health)
- Row panels use `"type": "row"` with `"collapsed": false`
- No separate ConfigMap creation needed in setup.sh for new dashboards â€” the loop handles it
- When adding panels to existing dashboards, start new panel IDs from 100+ to avoid conflicts
- KCP API groups filter: `tenancy.kcp.io|apis.kcp.io|core.kcp.io|topology.kcp.io|cache.kcp.io`
- Stacked area panels: `"stacking": { "mode": "normal" }` inside `custom`
- WATCH connections metric: `apiserver_longrunning_requests`; watch events metric: `apiserver_watch_events_total`
- Go exporter pattern: define struct â†’ register gauge(s) in var block â†’ MustRegister in init() â†’ implement collectX() â†’ call from collectMetrics()
- Reusable `condition` struct: `{Type, Status, LastTransitionTime, Reason, Message}` â€” used by shards, workspaces, etc.
- GaugeVec.Reset() before setting new values handles resource deletion (no stale label combos)
- KCP shards API: `/clusters/root/apis/core.kcp.io/v1alpha1/shards`
- KCP workspace types API: `/clusters/root/apis/tenancy.kcp.io/v1alpha1/workspacetypes`
- Boolean spec fields â†’ format as `"true"`/`"false"` strings for Prometheus label values
- KCP endpoint slices API: `/clusters/root/apis/apis.kcp.io/v1alpha1/apiexportendpointslices`
- Lifecycle metrics reuse existing workspace API response â€” no extra HTTP call; parse items as `workspaceLifecycleItem` for both phase counts and duration
- `calculatePhaseDuration()`: Ready phase uses Ready condition's lastTransitionTime; non-Ready uses creationTimestamp as fallback; clamp negatives with `math.Max(0, ...)`
- Stuck thresholds: Scheduling=5min, Initializing=10min, Terminating=15min â€” compare duration seconds vs threshold.Seconds()
- Recursive workspace discovery: `discoverWorkspaces(ctx, client, cfg, parentPath)` returns `[]*WorkspaceNode` tree; child path = `parentPath + ":" + childName`; only recurses into Ready workspaces
- `WorkspaceNode` is exported; `workspaceFullItem` is for discovery parsing (name, type, phase only)
- Per-workspace collection uses semaphore(10) + sync.WaitGroup for bounded concurrency; `collectForWorkspace()` queries 3 resource types per workspace
- Two separate ticker loops in main(): 30s for root-level `collectMetrics()`, 60s for per-workspace `collectWorkspaceMetrics()` + `collectTopology()`
- Discovery goroutine (5-min ticker) writes tree under Lock; collection goroutine reads under RLock â€” `sync.RWMutex` protects `workspaceTree`
- `collectWorkspaceMetrics` resets all per-workspace GaugeVecs before each cycle, then walks the tree setting info for all workspaces and collecting resources only for Ready ones
- Topology collection (`collectTopology`) walks the tree sequentially (not concurrent) because it accumulates consumer counts in a shared `map[exportKey]int`
- `apiBindingFullItem` has `spec.reference.export.path` (provider workspace) and `spec.reference.export.name` â€” separate from `apiBindingItem` which only has phase
- `exportKey{Workspace, Name}` struct as map key for aggregating consumer counts across workspaces
- Empty `identityHash` defaults to `"unknown"` for Prometheus label safety
- OTel async gauges: `Int64ObservableGauge` with `WithInt64Callback` reading from `exporterState` under RLock; periodic reader invokes callbacks at export time
- OTel import alias: `otelmetric "go.opentelemetry.io/otel/metric"` avoids conflict with `"go.opentelemetry.io/otel/sdk/metric"` (imported as `metric`)
- Dual-write pattern: `collectMetrics` writes Prometheus gauges + stores int64 counts in `exporterState` under Lock; OTel async gauges read state at export time
- OTel instruments gated by nil check: `otelInst *otelInstruments` is nil when OTEL_EXPORTER_OTLP_ENDPOINT not set; callers check `if otelInst != nil`
- Table panels: use `labelsToFields` + `organize` transforms to create clean columns from Prometheus metric labels
- Value mappings: `fieldConfig.defaults.mappings` maps numeric values to descriptive strings (e.g., 1 -> "Healthy")
- Gauge panels: set `min`/`max` in fieldConfig.defaults for scale range
- Unique count pattern in PromQL: `count(count by (label1, label2) (metric))`
- PrometheusRule CRDs need label `release: kube-prometheus-stack` for operator discovery
- AlertManager config in Helm values: `alertmanager.config.route/receivers` (not under alertmanagerSpec)
- Place `kubectl apply` for PrometheusRules in setup.sh after the Prometheus Helm install/upgrade block
- KCP Helm chart: `kcp.extraFlags` (list of `--flag` strings), `kcp.volumeMounts`, `kcp.volumes` for pod-level config
- Audit policy ConfigMap must exist before KCP Helm install â€” create it in setup.sh between namespace creation and helm install/upgrade
- KCP audit log path: `/var/log/kcp/audit.log` â€” shared via hostPath volume with OTel Collector filelog receiver
- OTel filelog json_parser: parsed fields go under `body.*`; use `body.field` in move operators, bracket notation for special-char keys (`body["annotations"]["kcp.io/path"]`)
- OTel Helm chart: `extraVolumes`/`extraVolumeMounts` are top-level values (not nested under `config:`)
